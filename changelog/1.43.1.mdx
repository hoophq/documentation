### TL;DR
- **New:** Runbooks v2 API with rules engine, multi-repo support, and new execution mechanisms; PATCH endpoint for partial connection updates; agent status filtering.
- **Improved:** Simplified runbook connection handling and better error handling across the platform.
- **Fixed:** Removed `access_mode` restrictions for runbooks; streamlined configuration management.

---

### New Features
**Next-generation runbooks and enhanced API capabilities.**

- **Runbooks v2 API with comprehensive execution capabilities**
  Runbooks v2 API supports multi-repository configuration with branch support, connection-level access control by user group and runbook rules, and normalized Git URL handling for SSH and HTTPS.
  [PR #1107](https://github.com/hoophq/hoop/pull/1107)

- **PATCH endpoint for partial connection updates**
  New PATCH endpoint allows partial connection updates without requiring full resource replacement.
  [PR #1110](https://github.com/hoophq/hoop/pull/1110)

- **Enhanced agent management with status filtering**
  New get-agent endpoint and status-based filtering in the list-agents endpoint for better agent monitoring.
  [PR #1109](https://github.com/hoophq/hoop/pull/1109)

---

### API Improvements
**Better documentation and endpoint management.**

- **Updated runbook API endpoints**
  Refactored runbook v2 rules and endpoints with improved API documentation. Removed the deprecated delete endpoint.
  [PR #1114](https://github.com/hoophq/hoop/pull/1114)

- **Improved error handling and response types**
  Improved error handling in agent management endpoints and added proper `AgentResponse` types.
  [PR #1109](https://github.com/hoophq/hoop/pull/1109)

---

### Bug Fixes & Refactoring
**Streamlined logic and improved reliability.**

- **Fixed runbook execution conditions**
  Removed `access_mode` requirements for running runbooks. Users can now run runbooks against specific connections while role-based access rules still apply.
  [PR #1123](https://github.com/hoophq/hoop/pull/1123)

- **Simplified runbook connection handling**
  Refactored connection persistence and loading logic, removed deprecated `read-string` usage, and improved local storage management.
  [PR #1123](https://github.com/hoophq/hoop/pull/1123)

- **Enhanced configuration management**
  Introduced `ConfigInput` type and streamlined configuration handling. Updated `GitHookTTL` from string to int and improved cache structure validation.
  [PR #1107](https://github.com/hoophq/hoop/pull/1107)

- **Removed deprecated connection validation**
  Removed deprecated validation for connection tag keys.
  [PR #1110](https://github.com/hoophq/hoop/pull/1110)

---

### Infrastructure
**CI/CD optimizations and workflow improvements.**

- **Streamlined release workflow**
  Removed unused CI jobs from the release workflow to cut build times.
  [PR #1113](https://github.com/hoophq/hoop/pull/1113)
