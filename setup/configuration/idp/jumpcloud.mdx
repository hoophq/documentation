<Info>
  JumpCloud does not emit JWTs as access tokens. The gateway validates authentication by making an HTTP request to the userinfo endpoint (OIDC spec).
</Info>

## Requirements

  - An [admin account in JumpCloud](https://console.jumpcloud.com)
  - `API_URL` is the public DNS name of the hoop gateway instance

Contact the hoop gateway administrator to retrieve the `API_URL` address.

## Identity Provider Configuration

<Steps titleSize="h3">
  <Step title="Create an Application">
  Log in at https://console.jumpcloud.com/

  * Go to **SSO** and click **Add New Application**
  * Select **Custom OIDC App** at the bottom of the page
  * Enter a name and optional description

  * In the **SSO** tab:
    * Add the Redirect URL: `{APIURL}/api/callback`
    * Add the Login URL: `{APIURL}/login`
  </Step>
  <Step title="Attribute Mapping">
  * Still in the **SSO** tab, select the standard scopes **Email** and **Profile**
  * (Optional) In the Groups attribute, select **Include groups attribute** and label it `https://app.hoop.dev/groups`
  </Step>
  <Step title="Collect the Credentials">
  * Click **Activate**
  * Copy both **Client ID** and **Client Secret**
  <Info>
  The secret cannot be retrieved later. Copy and store it now.
  </Info>
  </Step>
  <Step title="Collect Issuer Information">
    The issuer is fixed to `https://oauth.id.jumpcloud.com/`.
  </Step>
</Steps>

## Associating User Groups

To propagate groups to Hoop, create a new group:

  - Go to **User Groups > Create Group** and provide a name and description
  - Go to the **Users** tab and select the users to add to this group
  - Go to the **Applications** tab and select your custom OIDC app
  - Click **Save**
