## Requirements

  * A [GCP account](https://console.cloud.google.com)
  * `API_URL` is the public DNS name of the hoop gateway instance

Contact the hoop gateway administrator to retrieve the `API_URL` address.

## Identity Provider Configuration

<Steps titleSize="h3">
  <Step title="Create an Application">
    Log in at https://console.cloud.google.com/apis/credentials
    - Go to **Credentials > Create Credentials > OAuth Client ID**
    - In **Application type**, select **Web Application**
    - Give it a name (e.g., "Hoop")
  </Step>
  <Step title="Configure the Redirect URIs">
    - Click **Authorized redirect URIs** and add: `{API_URL}/api/callback`
    - Click **Create**
    - Copy the **Client ID** and **Client Secret**
  </Step>
  <Step title="Collect the Credentials">
    The credentials are available in the JSON file downloaded at creation time. They are also available at:

    - **Credentials > OAuth 2.0 Client IDs > Actions > Download**
  </Step>
  <Step title="Collect Issuer Information">
  The Issuer URI is `https://accounts.google.com`
  </Step>
</Steps>

## Configuring Groups

Groups sync via a request to the [Cloud Identity API](https://cloud.google.com/identity/docs/reference/rest/v1/groups.memberships/searchDirectGroups) as a best-effort operation.

<Note>
  This feature is available in version 1.35.2 and later.
</Note>

<Steps>
  <Step title="Add the Scope">
    Configure the gateway with the env **IDP_CUSTOM_SCOPES**:

    - `https://www.googleapis.com/auth/cloud-identity.groups.readonly`

    Users must grant consent for this access permission on first login to enable group synchronization.

    <Frame>
      <img src="/images/configure/identity-providers/google-authentication-consent.png" />
    </Frame>

    <Note>
      Restart the gateway after applying these changes.
    </Note>
  </Step>
  <Step title="Enable the Cloud Identity API in your project">

    - Visit the [Cloud Identity API Page](https://console.cloud.google.com/apis/api/cloudidentity.googleapis.com/overview)
    - Click **ENABLE**
  </Step>
</Steps>

<Warning>
  When configuring group synchronization, admin access may be revoked on your next sign-in.
  To maintain administrative privileges, set the `ADMIN_USERNAME` configuration parameter to a Google Workspace group you want to map as admin on Hoop.
</Warning>
