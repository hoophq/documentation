### Connection Setup
One-off commands can be executed inside a container using the `kubectl exec` command. The Web Console accepts inputs that are executed inside the container.
```bash
  hoop admin create conn k8s -a <agent> \
	-e KUBECONFIG=file://$HOME/.kube/config \
	-e KUBERNETES_RESOURCE=deployment/myapp \
	-- kubectl exec --stdin $KUBERNETES_RESOURCE -- bash
```
The command above will allow users to execute commands inside the specified Kubernetes resource. This is useful for running scripts inside a container.
### Command Line Usage Examples
```bash
  # execute 'env' inside the default container
  hoop exec k8s --input 'env'
  # execute 'ls -l' inside the default container
  hoop exec k8s -i 'ls -l'
```
