# Resource Roles

**A Resource Role is a named, secured connection that exposes a service to authorized users.** Admins define them; users access them through the gateway without touching underlying credentials.

## Types of Resources

Hoop supports three categories:

**Databases**: PostgreSQL, MySQL, and other database systems.

**Applications**: TCP forwarding, custom CLI tools, and web services.

**Custom Resources**: Specialized services with complex configurations and custom tool integrations.

## Key Features

**Access Control**: Group-based policies with approval workflows for sensitive resources.

**Security**: Encrypted communication, credential management, audit logging, and policy enforcement.

**Monitoring**: Activity tracking, usage analytics, and SIEM integration for compliance.
