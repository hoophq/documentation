**Set up a resource role in three steps:**

1. Resource Type Selection
2. Agent Configuration
3. Resource Role Credentials

## Resource Type Selection

Choose from available resource types. If unsure, start with Demo Postgres.

## Agent Configuration

Select the default agent or create a new one. The agent handles authentication, connection, and data transmission to the resource.

<Note>You can learn more about agents in the [Agents](/concepts/agents) section.</Note>

## Resource Role Credentials

Required fields vary by resource type:

<AccordionGroup>
  <Accordion title="Databases">
    - Host address (e.g. localhost)
    - Port number (e.g. 3306)
    - Username
    - Password
    - Database name
    - SSL mode (Optional)
  </Accordion>

  <Accordion title="Linux VM, Container or Console">
    ### Linux VM or Container
    - Access and manage with terminal commands
    
    ### Secure Shell Protocol (SSH)
    - Private Key (paste your file content)
    - SSH URI (ssh://[uri])
    
    ### Console
    - Ruby on Rails
    - Python
    - Node.js
    - Clojure

    Additional settings:
    - Operating system (MacOS/Linux)
    - Environment variables (Key-Value pairs)
    - Configuration files
      - Name (e.g. kub_config)
      - Content
    - Additional command (optional)
  </Accordion>

  <Accordion title="Network Service Access (ZTNA)">
    - TCP
    - HTTP (Coming Soon)

    Then provide:
    - Host address (e.g. localhost)
    - Port number
    - Agent selection
  </Accordion>
</AccordionGroup>

## Additional Configuration

After configuring credentials, set optional security features:

- Tags for organization
- **Access Requests**: Require approval before execution
- **Live Data Masking**: Mask sensitive data in query results

## Security Best Practices

<CardGroup cols={2}>
  <Card title="Meaningful Names" icon="tag">
    Name resource roles to indicate purpose and environment (e.g., `prod-postgres-ro`).
  </Card>

  <Card title="Tag Organization" icon="tags">
    Use tags to categorize and filter resource roles.
  </Card>

  <Card title="Enable Access Requests" icon="shield-check">
    Require approval for sensitive resources.
  </Card>

  <Card title="Data Protection" icon="mask">
    Enable Live Data Masking for resources that return PII.
  </Card>
</CardGroup>
