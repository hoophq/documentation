## What You'll Accomplish

AI Query Builder is a built-in AI assistant that helps you write queries and scripts. It understands your database schema and provides contextual suggestions, so you can:

- Get help writing complex SQL queries
- Generate scripts based on natural language descriptions
- Learn query syntax for unfamiliar databases
- Debug and optimize existing queries

---

## How It Works

When you enable AI Query Builder, Hoop sends your database schema (table names, column names, data types) to OpenAI's GPT-4o model. The AI uses this context to provide relevant suggestions.

<Note>
**Privacy:** Only schema metadata is sent to OpenAI—never your actual data. Queries are not associated with your identity or organization.
</Note>

### What Gets Sent to OpenAI

| Sent | Not Sent |
|------|----------|
| Table names | Actual row data |
| Column names | Query results |
| Data types | User identity |
| Your question/query | Company information |

## Quick Start

### Prerequisites

- Enterprise plan
- Gateway version 1.12.28 or higher
- OpenAI API key

### Step 1: Configure the Gateway

Set the `ASK_AI_CREDENTIALS` environment variable on your Hoop gateway:

```bash
ASK_AI_CREDENTIALS=sk-your-openai-api-key
```

For Kubernetes deployments, add to your gateway configuration:

```yaml
env:
  - name: ASK_AI_CREDENTIALS
    valueFrom:
      secretKeyRef:
        name: hoop-secrets
        key: openai-api-key
```

### Step 2: Restart the Gateway

After setting the environment variable, restart your gateway to apply the changes.

### Step 3: Use the AI Assistant

1. Open the Web App and connect to a database
2. Look for the **Ask AI** button in the bottom-right corner
3. Click it to open the AI assistant panel

## Using AI Query Builder

### Asking Questions

Type natural language questions and the AI will generate queries:

**You ask:**
> "Show me the top 10 customers by total order value"

**AI suggests:**
```sql
SELECT
  c.customer_id,
  c.name,
  SUM(o.total) as total_order_value
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_id, c.name
ORDER BY total_order_value DESC
LIMIT 10;
```

### Getting Help with Syntax

Ask about specific SQL features:

**You ask:**
> "How do I use a window function to get a running total?"

**AI explains:**
```sql
SELECT
  order_date,
  amount,
  SUM(amount) OVER (ORDER BY order_date) as running_total
FROM orders;
```

### Debugging Queries

Paste a query that isn't working:

**You ask:**
> "Why is this query slow?"
> ```sql
> SELECT * FROM orders WHERE customer_id IN (SELECT id FROM customers)
> ```

**AI suggests:**
> This query may be slow because of the subquery. Consider using a JOIN instead:
> ```sql
> SELECT o.* FROM orders o
> JOIN customers c ON o.customer_id = c.id
> ```

## Troubleshooting

### "Ask AI" Button Not Visible

**Check:**
1. Your organization is on an Enterprise plan
2. Gateway version is 1.12.28 or higher
3. `ASK_AI_CREDENTIALS` environment variable is set
4. Gateway was restarted after configuration

### AI Doesn't Understand My Schema

**Try:**
1. Be more specific about table names
2. Mention column names in your question
3. Describe the relationships between tables

### Slow Responses

OpenAI API response times vary. If responses are consistently slow:
1. Check your OpenAI API quota
2. Verify network connectivity from the gateway
3. Try simpler questions first

## Security Considerations

### Data Privacy

- **Schema only:** Only table/column metadata is sent to OpenAI
- **No data:** Actual row data is never transmitted
- **No identity:** Queries aren't linked to users or organizations
- **No storage:** OpenAI doesn't store your queries (per their API terms)

### API Key Security

- Store your OpenAI API key securely (use secrets manager)
- Use a dedicated API key for Hoop
- Monitor API usage for unexpected activity
- Rotate keys periodically
